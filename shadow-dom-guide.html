Web Components 実装ガイド: 素のJSとLitの比較

Web Components を理解し、Lit をマスターする

このガイドでは、まずブラウザの標準技術である Web Components を「素のJavaScript」で実装する方法を解説します。その上で、Lit がその実装をいかに簡潔で強力なものにするか、その「薄いラッパー」としての役割を明らかにします。

1. Web Components の標準実装（素のJavaScript）

Web Components は、カプセル化され再利用可能なUI部品を作るためのブラウザ標準技術です。ここでは、ライブラリを一切使わずにカウンターコンポーネントを実装してみましょう。

素のJavaScript版カウンター

実装コード (vanilla-counter.js)

class VanillaCounter extends HTMLElement {
  constructor() {
    super();
    // 1. Shadow DOM を作成し、外部から隔離された空間を確保
    this.attachShadow({ mode: 'open' });
    this._count = 0;
  }

  connectedCallback() {
    // 2. コンポーネントの見た目（DOM構造）を手動で構築
    this.shadowRoot.innerHTML = \`
      <style>
        .container { text-align: center; padding: 1em; border: 1px solid #ddd; }
        .count { font-size: 2em; margin: 0.5em 0; }
        button { font-size: 1em; padding: 0.5em 1em; cursor: pointer; }
      </style>
      <div class="container">
        <h3>素のJavaScript版</h3>
        <div class="count">カウント: 0</div>
        <button>クリック</button>
      </div>
    \`;

    // 3. DOM要素への参照を取得
    this._countDisplay = this.shadowRoot.querySelector('.count');
    const button = this.shadowRoot.querySelector('button');

    // 4. イベントリスナーを手動で設定
    button.addEventListener('click', () => {
      this._count++;
      // 5. データが変更されたら、DOMを手動で更新
      this._countDisplay.textContent = \`カウント: \${this._count}\`;
    });
  }
}

customElements.define('vanilla-counter', VanillaCounter);

標準実装のポイント

  • 手動でのDOM構築: innerHTML や createElement を使って、コンポーネントのHTML構造を文字列や命令型コードで組み立てる必要があります。
  • 手動でのDOM更新: データ（例: _count）が変更された後、表示を更新するために手動でDOM要素（例: _countDisplay）を見つけて、その内容を書き換える必要があります。
  • 命令的なコード: 「ボタンがクリックされたら、この要素のテキストをこう変えなさい」というように、手順を一つずつ記述する必要があります。
重要: これがブラウザがネイティブでサポートしている Web Components の基本的な姿です。Lit を理解するには、まずこの基本形を把握することが不可欠です。

2. Lit は、この標準実装をどうラップするか？

Lit は、前述の面倒な手作業を自動化し、開発者が「何をしたいか」だけに集中できるようにする「薄いラッパー」です。同じカウンターを Lit で実装すると、コードは以下のようになります。

Lit版カウンター

実装コード (lit-counter.js)

import { LitElement, html, css } from 'lit';

class LitCounter extends LitElement {
  // 2. スタイルを宣言的に定義 (CSS in JS)
  static styles = css\`
    .container { text-align: center; padding: 1em; border: 1px solid #667eea; }
    .count { font-size: 2em; margin: 0.5em 0; }
    button { font-size: 1em; padding: 0.5em 1em; cursor: pointer; background: #667eea; color: white; border: none; }
  \`;

  // 5. リアクティブプロパティを定義 (データとUIが自動で同期)
  static properties = {
    count: { type: Number }
  };

  constructor() {
    super();
    this.count = 0;
    // 1. Shadow DOM は LitElement が自動で作成
  }

  // 3. HTMLテンプレートで見た目を宣言的に定義
  render() {
    return html\`
      <div class="container">
        <h3>Lit版</h3>
        <div class="count">カウント: \${this.count}</div>
        // 4. イベントも宣言的に記述
        <button @click=\${() => this.count++}>
          クリック
        </button>
      </div>
    \`;
  }
}

customElements.define('lit-counter', LitCounter);

素のJS vs Lit: ラッパーとしての役割

Lit が「素のJS」のどの部分を、どのようにラップ（簡略化）しているのかを比較してみましょう。

素のJavaScript（標準API） Lit（薄いラッパー） Litが自動化・簡略化すること
this.attachShadow(...) class extends LitElement ✅ Shadow DOMの自動作成
innerHTML = '<style>...</style>' static styles = css`...` ✅ スタイルの注入と最適化
innerHTML = '<div>...' や createElement() render() { return html`...` } ✅ 宣言的なテンプレートによるDOM構築
addEventListener('click', ...) @click=\${...} ✅ イベントリスナーの自動設定・解除
element.textContent = ... this.property = ... ✅ データの変更を検知し、DOMを自動で更新（リアクティビティ）

結論: Litは「宣言的なUI」と「リアクティビティ」を提供する薄い層

Litは、Web Components の標準的な手続き（DOM作成、更新、イベント設定）を内部で処理してくれます。これにより、開発者は命令的なコードから解放され、**「データ（プロパティ）がこうなれば、見た目（テンプレート）はこうなる」という宣言的な記述に集中できる**のです。

  • Lit は魔法ではありません。Web Components の上に成り立つ便利なツールセットです。
  • Lit を学ぶことは、Web Components の標準をより効率的に活用する方法を学ぶことと同じです。

深掘り: Shadow DOM の主要な概念

Lit の背景にある Shadow DOM の技術について、もう少し詳しく見てみましょう。

Shadow Host, Shadow Root, Shadow Tree

Shadow DOM は主に3つの要素で構成されます。

  • Shadow Host: Shadow DOMがアタッチされる通常のDOM要素（例: <lit-counter>）。
  • Shadow Root: 隔離されたDOMツリーの起点となるノード。element.shadowRootでアクセスできます。
  • Shadow Tree: Shadow Root以下のDOM構造全体。この内部のスタイルや要素は外部から完全に隔離されます。

カプセル化: スタイルの漏洩を防ぐ

Shadow DOM の最大の利点はスタイルのカプセル化です。コンポーネント内部で定義したCSSは、そのコンポーネントの外に影響を与えず、また外部のスタイルがコンポーネント内部に影響することもありません。

通常のDOM（Light DOM）

<style> p { color: blue; } </style>
<p>この文字は青色</p>

Shadow DOM

<style> p { color: blue; } </style>
<my-component>
  #shadow-root
    <style>p { color: red; }</style>
    <p>この文字は赤色</p>
</my-component>
